Common Docker & NestJS Errors

=========================================================
Error: Module not found: Can't resolve '@nestjs/config'
=========================================================
Symptoms
ERROR in ./apps/auth/src/auth.module.ts 13:17-42
Module not found: Error: Can't resolve '@nestjs/config' in '/usr/src/app/apps/auth/src'
TS2307: Cannot find module '@nestjs/config' or its corresponding type declarations.
Root Cause
Docker's build cache was preventing npm install from running, so the node_modules were not being installed into the container's anonymous volume. When the container started, the dependencies were missing.
Solution
Rebuild all services without using Docker's cache:
docker-compose down
docker-compose build --no-cache
docker-compose up
The --no-cache flag forces Docker to:

Skip cached layers
Actually execute RUN npm install
Properly install all dependencies into the anonymous volume /usr/src/app/node_modules

Why This Works
Your docker-compose volume configuration is correct:
yamlvolumes:
- .:/usr/src/app                                                                                                                                                                                                        # Mount host directory
- /usr/src/app/node_modules                                                                                                                                                                                             # Anonymous volume protects node_modules
The anonymous volume /usr/src/app/node_modules prevents the host directory mount from overwriting the container's installed dependencies. However, if Docker's cache skips the install step, this volume remains empty.
Prevention

Use --no-cache after adding new packages to package.json
For normal development, docker-compose up works fine once dependencies are built
If issues persist, ensure @nestjs/config is in your root package.json: "@nestjs/config": "^4.0.2"
